## Release v1.2.0 — the Furnace, multi-CLI adapters, and a hardened core
*Released at 2026-07-03T04:16:05.330Z*

### Highlights

- **The Furnace** — an overnight autonomous ticket runner. Curate a magazine of burn-approved tickets and it stokes them unattended (implement → review → reimplement → leave a PR at Ready, never merging), driven from a board-anchored drawer you drag tickets into. Burn-rate control, watchdog + circuit breaker, graceful failure handling, park-as-swimlane, real `gh` PR approvals, token-exhaustion pause/retry, and rate-limit cooldowns.
- **Multi-CLI adapter layer** — every Claude-only surface genericized into a CLI-agnostic layer routed per framework (Claude, Copilot, Gemini), guarded by a ratcheting adapter-boundary CI check and a cross-adapter contract test net.
- **Hardened core** — a unified hardened git/gh runner (timeout + non-interactive env + kill-tree) routed across every git/gh call site (epic FLUX-996), plus worktree fail-closed safety, dev/sync-watcher hardening, and SSE heartbeats.
- **MCP surface** — tool surface consolidated 33→~23 (breaking), resources + templates, structured output, and richer server instructions/annotations.
- **Board perf & UX** — slimmer list payload, no terminal-ticket polling, no remount stall on view switches, redesigned agent-management window, and Visual Recap artifacts on Ready.

See [`CHANGELOG.md`](../../CHANGELOG.md) for the curated, themed breakdown. Full ticket manifest below.
