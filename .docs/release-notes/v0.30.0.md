# Release v0.30.0

*Released 2026-06-16*

The largest release to date. Three big pillars — **concurrent multi-agent isolation via git worktrees**, a **cross-worktree diff & changes viewer**, and a deep pass on **agent context-window efficiency** — plus a clutch of reliability fixes (settings no longer revert, builds never serve stale code).

## 🌳 Per-branch git worktree isolation (FLUX-516)

Task agents no longer share one working directory. Each agent now runs in its **own git worktree on its own branch** (`.eh-worktrees/<repo>-<id>`), with an independent HEAD and working files but a shared `.git` object store. This fixes the core incident where starting one task flipped another session's branch out from under it (a CLI session's commit landing on a concurrently-started branch can no longer happen). Two agents on different branches — and a human/CLI session on `master` — now run side-by-side without collisions.

- **FLUX-517** — core lifecycle engine: create / remove / prune, one-branch-one-worktree guard (never `--force`), configurable concurrency cap, and a dirty-safe **detach** escape hatch that surfaces uncommitted work onto `master` instead of discarding it.
- **FLUX-518** — junctions the main tree's `node_modules` (root + engine + portal) into each worktree so it builds without a reinstall.
- **FLUX-519** — spawns each agent with `cwd` = its worktree (the two-roots split: engine workspace root vs per-task execution root).
- **FLUX-520** — ticket state stays canonical: a worktree's own `.flux` copy is never read.
- **FLUX-521** — opt-in everywhere: a settings default (off), a per-launch portal choice, an MCP `worktree` param, and a Detach control.
- **FLUX-522** — one-click "Start in worktree" opens a fresh VS Code window already rooted in the worktree.

## 🔍 Cross-worktree diff & changes viewer (FLUX-527)

A new top-level **Changes** view that aggregates every in-flight change across all execution roots in one read-only surface: each active worktree (committed-ahead + uncommitted + untracked), the **main tree's loose uncommitted work**, and a **"Recently merged"** section of finished tickets. Click any file to see its diff.

- **FLUX-528** — engine diff-aggregator + `GET /api/diffs/overview` / `/diffs/file`, running `git -C <worktree>` per worktree (so a worktree's *uncommitted* changes are actually visible).
- **FLUX-529** — **collision radar**: files touched by more than one worktree (or also loose on `master`) are flagged as imminent merge conflicts — something only EH can see, because it knows about every worktree at once.
- **FLUX-530 / FLUX-531** — the Changes panel, focus/status/path filters, an "Exclude done" toggle, and board click-throughs (a card "Diffs" button, the change-count badge, the worktree chip, and a context-menu "View changes" all jump straight in).

## 🎛️ Worktree board UX (FLUX-533)

Worktrees are now first-class on the board: a **filter chip** to isolate the board to a specific worktree (with per-worktree changed-file counts), an **inline detach** + change badge on cards, and a **redesigned right-click menu** — phase-contextual actions, viewport-clamped side-flyout submenus, and fuzzy "Attach to parent/branch/worktree" pickers.

## 🧠 Agent context-window efficiency

A focused effort to keep agent context cheap as tickets age:

- **FLUX-501** (phases **FLUX-503/504/505/506/507**) — **collapse old comments in the agent digest**: summarized history entries collapse to a one-line summary with expand-on-demand (`get_ticket(expand:[ids])`), while pinned and user-authored entries stay full. Humans still see everything in the portal. A heavily-worked ticket's `get_ticket` payload drops dramatically.
- **FLUX-490 / FLUX-500** — per-phase / per-persona **MCP server profiles** at launch (give a grooming agent fewer servers than an implementer) with a portal editor.
- **FLUX-488** — MCP **response-size instrumentation** + a portal debug view to see exactly where the context budget goes.
- **FLUX-498 / FLUX-499** — leaner `get_ticket` (omit the body already echoed in the launch prompt) and drop `status_change` / routine activity noise from the digest.

## 🐛 Fixes & reliability

- **FLUX-532** — **settings no longer revert or leak**. In orphan-storage mode `config.json` / `read-state.json` were committed to the synced `flux-data` branch, so a sync fetch reverted local settings and leaked board structure between clones. They're now local-per-workspace (gitignored in the store; already-tracked copies are migrated out, files preserved).
- **FLUX-496** — build reliability: stale compiled JS in `src/` and version-keyed SEA extraction could serve old code; the build now fails on stale artifacts and forbids `tsc` emit.
- **FLUX-502** — the diff viewer now works for branch-less (generic) tickets.
- **TRAIL-1** — a reused worktree self-heals its `node_modules` junctions.
- **FLUX-508** — removed stray file-locking debug scratch from the repo root (+ a guard against re-adding it).
- **FLUX-445** — code-quality cleanup from the swimlane review.

## 📚 Docs & onboarding

- **FLUX-514** — a seeded **demo workspace** + README showcase (GIF shot-list & recording guide) so a first-time visitor sees the product in motion.
