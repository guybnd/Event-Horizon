## Release v0.8.1 — Workflow Builder & Multi-Agent Sessions
*Released 2026-05-26*

### Highlights

This release adds the **Workflow Builder** for visually composing agent workflows from file-backed skills, extends the session store for **multiple concurrent agent sessions**, and fixes several portal and MCP reliability issues.

---

### Features

- **Workflow Builder UI** (FLUX-312): Visual workflow editor with drag-and-drop skill composition, backed by `.docs/skills/` markdown files. Includes new agent and workflow engine models with REST routes.
- **Multi-agent session store** (FLUX-283): Session store now supports multiple concurrent agent sessions, enabling parallel agent execution without session conflicts.
- **Epic cards**: New epic card component in the portal task board for grouping related tickets visually.
- **Curated release notes CI**: GitHub Actions release workflow now extracts the release body from `.docs/release-notes/`, enabling hand-crafted changelogs in GitHub Releases.

### Bug Fixes

- **Notification bell for Ready/Done transitions**: Fixed notification bell not firing when tickets transitioned to Ready or Done status.
- **Copilot.exe MCP tool loading** (FLUX-310): Prefer `copilot.exe` binary over `node + entrypoint` to ensure MCP tools load correctly on Windows.
- **Unread count badge and modal overlay**: Fixed unread count badge not updating and full-view modal overlay z-index issue.

---

### Upgrade Notes

- No breaking changes from v0.8.0.
- The workflow engine routes are additive — existing API consumers are unaffected.
- Session store changes are backwards-compatible; existing single-session workflows continue to work.
