## Release v0.5.0
*Released on May 13, 2026*

This release focuses on stability improvements, Windows compatibility, and enhanced git sync capabilities for orphan branch storage mode.

### 🪟 Windows Compatibility

- **Fixed Windows agent spawn errors** — Resolved EINVAL errors and missing prompt arguments when launching Claude Code sessions on Windows
- **Improved Windows compatibility across the codebase** — Fixed path handling, environment detection, and command execution on Windows systems
- **Fixed npm prefix lookup** — Corrected command execution to properly locate global npm packages on Windows

### 🔄 Git Sync & Storage

- **Enhanced orphan branch sync reliability** — Prevented ticket ID collisions and data loss during sync operations
- **Added conflict resolution UI** — New interface for resolving merge conflicts when syncing tickets stored in orphan branch mode
- **Fixed sync race conditions** — Now fetches latest changes before pushing to prevent lost updates
- **Added sync status indicator** — Real-time sync status now visible in the portal UI
- **Manual sync trigger** — Click the sync status button to force an immediate sync
- **Fixed ticket routing on startup** — Tickets are now correctly written to `.flux-store/` when worktree is attached

### 🐛 Bug Fixes

- **Fixed engine crashes** — Resolved ES module compatibility issues that caused the engine to crash
- **Fixed new project setup** — New projects now properly initialize with documentation
- **Fixed environment detection** — Engine no longer crashes when agent is not configured in the environment
- **Fixed workspace sync watcher** — Workspace route now properly restarts the sync watcher after activation
- **Fixed dragging tickets between columns** — Board view no longer resets when dragging tickets

### ✨ UI & UX Improvements

- **Redesigned Agent Session interface** — Cleaner, more intuitive UI for monitoring active agent sessions
- **Auto-stop CLI sessions** — Agent sessions now automatically stop when a ticket moves to Ready status
- **Show corrupted ticket indicator** — Portal now displays a warning icon for tickets with invalid frontmatter
- **Enforced completion comments** — System now requires a descriptive comment when moving tickets to Ready status

### 🧪 Developer Experience

- **Added body editing to patch-ticket CLI** — New `--body` and `--body-file` flags for updating ticket descriptions programmatically
