## v0.27.2

Fixes two critical bugs in the v0.27.1 Windows SEA build that broke MCP mode — the channel every agent uses to connect (`event-horizon.exe --mcp`). Both only manifested at runtime, so the green CI build masked them.

### Fixes

- **MCP server failed to load in the SEA binary.** Inside a Single Executable Application the global `require()` is Node's embedder require, which resolves built-in modules only and cannot load a file from disk. Loading the extracted `mcp-server.js` now uses `module.createRequire()`, which provides a filesystem-capable require.
- **`SEA assets not yet extracted` crash in MCP mode.** `mcp-server.js` is a separate bundle and carries its own instance of the packaged-mode module, so its extract-dir singleton was never initialised by `index.js`. `getSeaExtractDir()` now re-derives the deterministic extract path (`tmpdir/event-horizon-<version>`) when its local singleton is unset.

Verified end-to-end on Windows: the SEA binary now completes the MCP handshake and lists all tools. Windows Defender does not flag the SEA build (the v0.27.0 pkg build was flagged as `Trojan:Script/Wacatac.C!ml` on the same machine).
