## Release v1.9.0
*Released at 2026-07-21*

> The fresh-install reliability release. v1.9.0 closes out the entire class of "installed it, typed a message, nothing ever happened" first-run failures reported on macOS — from binary resolution through auth to hung spawns — and dresses the app in its new Axis Mundi identity.

### Fresh-install & first-run reliability

**No more silent wedges.** A first chat message could spawn an agent CLI that hung forever with zero output (e.g. `claude` installed but never logged in — headless mode just blocks). The session sat `running` permanently: no reply ever, and every retry hit "Orchestrator/Furnace session already active" or "Task already has a live CLI session" until the app was restarted. Three-layer fix: the chat now queues a message into an in-flight turn instead of cold-starting a duplicate that 409s (FLUX-714), a silent-spawn watchdog kills any spawned agent that produces no output for 3 minutes and explains what to check (FLUX-1596), and a crashed board/Furnace turn now surfaces as a visible ⚠️ error in the chat instead of failing invisibly (FLUX-1596).

**The right `claude`, every time (macOS).** Bare `spawn('claude')` could resolve a different, stale binary than the one your terminal uses — each with its own credential store, producing a persistent 401 while "the terminal works fine." The engine now resolves the binary through your login shell, the same way your terminal does (FLUX-1600, from the FLUX-1597 field report).

**Auth failures diagnose themselves.** On a 401/403 the engine now runs the checks a support thread would ask you to paste by hand — which binary was spawned vs. what your shell resolves, duplicate PATH installs, `ANTHROPIC_API_KEY`/`apiKeyHelper`/base-URL shadowing in settings or env — and shows a structured verdict card in the chat with the specific remedy (and a copy button for `claude login`) instead of a raw provider error (FLUX-1599, FLUX-1601). After re-login it detects the refreshed credentials and recovers automatically. Sessions also fail fast on auth errors — aborting on the first 401/403 instead of riding out 10 blind retries.

### Axis Mundi

**New identity, everywhere.** The Axis Night / Axis Day themes with lapidary display type and the axis-mark logo; Axis Day is the default theme for fresh installs (FLUX-1602). The mark is wired through the full surface area — app/tray/window icons, the Windows exe, and the portal favicon (FLUX-1589).

### UX

**New Ticket, redesigned.** The create screen was rebuilt — cleaner layout, deduplicated grooming CTA, and a right-sized description box (FLUX-1592).

**One Stop button.** While a session is live, the board card's action cluster collapses to a single Stop control — no more accidental double-dispatch from a crowded card (FLUX-1603).

**Chats stay on their board.** Floating chat windows are scoped to the workspace they were opened in, so switching boards doesn't strand or cross-wire them.

### Engine & agent hygiene

- Managed-ticket upserts resolve against the requesting workspace instead of the ambient one — a multi-board correctness fix (FLUX-1579).
- The grooming skill module got a final compression pass — each constraint stated once, symbol anchors over line numbers (FLUX-1583, FLUX-1584, FLUX-1585).
- The sync watcher's conflict-resolution git calls now run through the hardened git runner (timeout, non-interactive env, tree-kill) — closing the last bare `git push` that could hang the sync path.
- The FLUX-850 hard-gate on skip-permission sessions auto-advancing past Ready was reverted pending a less disruptive design.
