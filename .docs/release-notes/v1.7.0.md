## Release v1.7.0
*Released at 2026-07-17T07:19:42.600Z*

### Highlights

**Plan review & HITL hardening.** Clicking a stale Approve button no longer hangs the portal — the approve write fast-fails on the engine when the verdict was already consumed, the client fetch is timeboxed, and the auto-gate now broadcasts its consume so stale cards tear themselves down (FLUX-1485). Auto-approved plans now still notify you that a plan file exists to read (FLUX-1363), every plan-review surface gained an "Approve & start" action (FLUX-1369), and the Needs-You card and reference mode got a usability pass (FLUX-1472, FLUX-1475).

**Furnace redesign.** The Furnace is now a floating overlay anchored to the Done column with a fire-orange visual hierarchy pass (FLUX-1487), plus larger touch targets on its corner caret (FLUX-1337).

**Performance & reliability.** Fixed the multi-second lag when swapping workspaces (FLUX-1465), the laggy worktree task-start selector (FLUX-1464), and cards not loading on cold boot until a refresh (FLUX-1460).

**Design system.** Portal state colors are unified under semantic `--eh-state-*` tokens (FLUX-1478) and action rows share one Button component (FLUX-1477), alongside a broad polish batch: chat transcript/composer cleanup, artifact pane header merge, chat-dock chip strip fades, resizable windows from every corner, and stripe/Esc-key fixes (FLUX-1473, FLUX-1474, FLUX-1476, FLUX-1461, FLUX-1415, FLUX-1352, FLUX-1463).

**Leaner agent prompts.** MCP tool descriptions went on a diet — behavioral contracts stay in schemas while lore moved to on-demand `read_skill('tools')` guides (FLUX-1468), agents can pull skill modules themselves via a new `read_skill` tool (FLUX-1466), launch focus is now dynamic facts instead of persisted boilerplate (FLUX-1469), persona playbooks dedupe against skill modules (FLUX-1471), skill modules declare their delivery surface (FLUX-1480), and role derivation is unified into one phase→persona layer across solo chats, dispatched sessions, and delegation (FLUX-1226).
