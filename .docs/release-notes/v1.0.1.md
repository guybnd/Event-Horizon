## Event Horizon 1.0 — an IDE for the agent era (1.0.1, revamped)

Event Horizon is the development environment for the **agent side** of your work: break work into tickets, hand each to an agent that carries full context and runs on its **own git branch, in parallel**, and review everything as a **real pull request** before it lands. Tickets, plans, and history are plain markdown in your repo — local-first, no cloud, no account. It orchestrates an agent CLI you already use (Claude Code, Gemini CLI, GitHub Copilot CLI) or a supported IDE; it ships no LLM of its own.

**1.0.1 is the 1.0 release, hardened.** It's the same major release — now with a batch of reliability fixes from real day-one use. If you're new here, the full 1.0 story is below; if you're already on 1.0.0, here's what changed.

### New in 1.0.1

A focused round of reliability and durability fixes on top of 1.0:

- **Durable, resumable human-in-the-loop prompts.** A *Require Input* question now survives an engine or session restart and no longer hits the old ~300s ceiling — the prompt is persisted (async, coalesced writes), re-bound to its session, and kept out of the flux-dir/sync watchers so it can't churn your data branch (#141, #156, #145, #155).
- **No more orphaned processes.** Stopping an agent now **tree-kills the whole process tree**, so its MCP servers get reaped instead of leaking stale `node` processes in the background (#163). Completed sessions also stop showing as forever-"Working" with a runaway timer in the active-sessions panel (#149).
- **Cheaper, safer orchestration.** `delegate_parallel` no longer re-launches the entire fleet (~3× token cost) when the MCP transport drops right after spawn (#144). A delegate double-launch race is closed with a pre-spawn reservation (#146), reopen dispatch re-fires correctly on a modal switch (#147), and dispatched-session output now **streams live to the board**, not just the ticket transcript (#154).
- **Isolation by default.** Agent dispatch now runs in an isolated worktree by default through a single canonical code path, hardened with a race guard and unit tests (#150, #159). Serena binds to the **per-task worktree** (unique project name) so symbol edits land in the right tree, never on `master` in the main checkout (#143).
- **One loud "needs you" surface.** Require-input tickets are unified into a single Pending bar with a persistent badge on the chat tab and window polish — auto-pop, dismiss, close-on-jump, and a taskbar require-input flag (#160, #161, #162).
- **Board orchestrator cold-resume.** Opening a cold board now offers **"Resume conversation" vs "Start fresh,"** re-priming bounded to the last N turns so it stays fast (#155, #142, #158).
- **Two new verbs.** **Merge** — fold several chats/tickets into one effort (#151). And an **engine-side branchless finish** (zero-token) via `POST /:id/finish` (#152).
- **UI polish.** Ticket-modal diff and session-run panels default to collapsed (#153), accessibility passes on the chat orchestration surfaces (#148), and the floating panel no longer swallows minimize/close clicks while dragging (FLUX-836).

### Event Horizon 1.0 — the highlights

- **A board of agents, not one chat** — many tickets in parallel, each a persistent agent chat with full context. Minimize it to the dock and the session keeps running.
- **Isolated parallel runs** — every ticket gets its own git worktree + branch, so agents work at the same time with zero file collisions.
- **Inline PRs** — moving a ticket to Ready pushes the branch and opens a PR; `finish` squash-merges it to Done.
- **Orchestration** — specialist personas, scatter-gather under a reviewing supervisor, and a board orchestrator that triages your whole project into tickets.
- **Local-first & yours** — markdown in your repo (or an orphan data branch via Git Sync); works offline, travels with a `git clone`.
- **Multi-repo groups, board-driven release notes, and an opt-in desktop app** (its own window + tray; `.dmg`/`.exe` below).

### Hardened for 1.0

- Loopback-only binding with `Host`/`Origin` guards — a page you merely visit can't drive the API. Opt into LAN exposure with `EH_ALLOW_REMOTE=1` (no auth — trusted networks only).
- Atomic writes + corruption guards for `config.json` and tickets; a malformed `.mcp.json` no longer clobbers your other MCP servers; file-watcher errors degrade gracefully instead of crashing the engine.
- First-run identity/attribution fixed, onboarding wizard + "Bootstrap with AI", overhauled Settings and notifications, and a responsive board for wide displays.

### Install

- **Desktop app** — download the `.dmg` (macOS, Apple Silicon) or `.exe` (Windows) below: Event Horizon in its own window with a taskbar/dock entry.
- **Tray binary** — a single executable that runs in the system tray; no Node required.
- **Run from source** — `event-horizon-source.zip` below; Node 20+.
- **Bring your own agent** — Claude Code, Gemini CLI, or GitHub Copilot CLI, or a supported IDE (Cursor, Windsurf, Cline).

> **First launch:** the builds are unsigned, so macOS Gatekeeper and Windows Defender may show a first-launch false positive. **macOS:** `xattr -d com.apple.quarantine <path>`, or right-click → Open. **Windows:** allow the file in Defender, or run from source. Full details in the README's first-run troubleshooting.
