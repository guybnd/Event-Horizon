## Release v1.8.1
*Released at 2026-07-19T15:54:06.199Z*

> v1.8.1 is the first installable release of the 1.8 line — v1.8.0's release pipeline was blocked by its engine-test gate before artifacts were built (test-infrastructure drift, no product defect; fixed in FLUX-1581). Everything below under "What's in 1.8" shipped in the v1.8.0 codebase and is delivered by these artifacts.

### New in 1.8.1

**Release pipeline restored.** Fixed the four engine-test failures that blocked v1.8.0's artifact build: multi-workspace isolation tests now compare against canonical workspace roots (8.3/symlinked temp dirs), the persona-model spawn tests stub the CLI binary probe on runners without `claude` installed, a `child_process` test mock gained its missing `exec` export, and a drifted ignore-list fixture now derives from the engine's single source of truth (FLUX-1581).

### What's in 1.8

**True multi-workspace.** The headline of this release: one engine now serves N boards at once. Every layer went per-workspace — the workspace registry and live-board bootstrap (FLUX-1446, FLUX-1529), per-request HTTP/SSE routing via `X-EH-Workspace` (FLUX-1530), sync workers, write locks, timers, Furnace batches, notifications and HITL prompts (FLUX-1453, FLUX-1451, FLUX-1452, FLUX-1513, FLUX-1555), and CLI sessions tagged with their owning board (FLUX-1531). The portal switches boards non-destructively from a workspace tab strip — no reload, no killed sessions (FLUX-1454, FLUX-1455). Background loops (Temper, PR reconcile, gate runner) and engine self-dispatch now bind to the workspace that owns the work instead of whichever board was last opened (FLUX-1548, FLUX-1551, FLUX-1554), and the board orchestrator + Smelter chat each get a persistent session per workspace — switch, don't reset (FLUX-1580).

**Boot & runtime performance.** Cold boot went from ~60s on a 2000-ticket board to seconds via a parallel scan, persistent index, and two-phase load (FLUX-1547, FLUX-1540). The idle board no longer burns ~24 main-thread frames/sec (FLUX-1552), motion-batch re-render amplification is fixed (FLUX-1542), and interaction-burst render costs came down across cluster cards and token pills (FLUX-1553, FLUX-1549).

**Apple-feel polish, wave 2.** Ten polish rooms from the responsiveness audit (FLUX-1516): shared motion physics tokens with card→modal morph and card flight (FLUX-1507), optimistic commit-first actions with patch-first SSE (FLUX-1505), ghost session placeholders and a skeleton system over spinners (FLUX-1506), depth-stacked modals (FLUX-1518), staggered cold-boot reveal (FLUX-1519), rolling counters (FLUX-1520), toasts that fly from and dismiss into their card (FLUX-1522), interruptible retargetable springs (FLUX-1523), preserved scroll/selection across views (FLUX-1524), drag lift physics (FLUX-1525), and one success choreography — confetti is reserved for releases (FLUX-1526).

**Cheaper, smarter orchestration.** Batch grooming lets one session groom up to five sibling tickets in a sitting (FLUX-1383), each phase's dispatched lead now carries a deliberately trimmed toolset (FLUX-1462), the Scratchpad persona gained phase handoff and per-phase model overrides (FLUX-1479), and a communication-style contract makes agent output concise and reader-first across personas and skill modules (FLUX-1502). A new token observability panel surfaces agent-payload metrics and waste signals — including a digest-savings counterfactual and a board-wide ranked cost view (FLUX-1512).

**Safety rails.** A skip-permissions dispatched session can no longer auto-advance a ticket past Ready without surfaced confirmation (FLUX-850), finish/merge can gate on your repo's own CI checks, stack-agnostically (FLUX-560), ticket writes gained compare-and-swap so stale editors can't silently clobber concurrent changes (FLUX-1550, FLUX-1534), the plan gate defaults to Auto → You on fresh boards (FLUX-1497), and a real end-to-end lifecycle integration test now exercises grooming → Ready (FLUX-1569). The desktop app also warns before closing while sessions are still running (FLUX-1541).

**Chat & ticket UX.** Edit-and-resend / retry a chat turn (FLUX-685, FLUX-1566), fixes for vanishing or double-dispatched chat messages (FLUX-1493, FLUX-1494, FLUX-1495), the board orchestrator's flat 3s spawn tax removed (FLUX-1496), per-field dirty/instant-save editing on the ticket sideview (FLUX-979, FLUX-1568), and the sideview's section stack replaced by a tab bar with availability badges (FLUX-1515).
