# Release v0.51.0

*Released 2026-06-19*

The release that lays the **foundation for treating the board as a curated lens over an immutable conversation**. v0.51.0 lands the keystone architecture split — raw turns become append-only and addressable, the board becomes re-derivable — and ships the first **orchestrator triage ritual** on top of it: propose a batch of restructurings, approve them in one pass, never mutate the board silently. Around that, the per-ticket **chat keeps maturing into a real editing surface** — inline per-edit diffs, image paste, and reliable id chips — and the board itself learns to **fold epics like it already folds PRs**.

## 🧱 Substrate vs projection — the foundation everything else rests on

**FLUX-658** lands the keystone split from the conversation-curation epic (FLUX-601): *the raw turns are an immutable, append-only substrate; the board is a re-derivable lens over them.* This lifts the existing "history is append-only, never delete" rule from per-ticket to **whole-board**, and it's what makes the upcoming extract / merge / archive / board-rebase verbs *safe* — they become re-projections, not destructive edits.

- The JSONL transcript sidecar is now the **substrate of record** — no new store invented; the existing ticket-history projection is reframed as one derived view.
- Every appended turn is wrapped in a **stable envelope** (`{ turnId, streamId, seq, ts, role, raw }`) with a monotonic per-stream sequence, so turns are finally **addressable and sliceable** (`readTurns` / `sliceTurns`) instead of being reachable only by implicit line number — legacy un-enveloped transcripts still read losslessly.
- A pure `projectTranscript(turns, ops?)` **re-derives the chat view from the substrate alone**, proving the board is a *function of* the turns (no user-visible change today), with the curation op-log seam shaped in from day one for the verbs that come next.

## 🧭 Orchestrator triage teeth + the board-rebase ritual

**FLUX-659** gives the board orchestrator real teeth: it can now **propose a batch of restructurings and have you approve them in one pass** — *"promote these 2 chats to cards · fold this stream into FLUX-X · archive the rest · leave the others"* — instead of silently reorganizing the board.

- New **`propose_board_rebase`** MCP tool emits a structured batch that **parks engine-side and broadcasts over SSE** — it never mutates anything on its own (the same park-and-broadcast shape as the FLUX-605 permission flow, but batched and fire-then-resolve).
- A **board-rebase approval panel** in the orchestrator dock renders one row per proposed item with per-item toggles (default-checked), summary + rationale, and a single **Apply approved** / **Dismiss**. Approved items run through a verb registry (`status` / `archive` / `dispatch` / `leave` live; `promote` / `fold` register as their verbs land).
- **Teeth, not just prose:** the restructuring verbs (`archive_ticket`, `extract_ticket`, `merge_tickets`) are added to the **confirm-permission tier**, so a direct mutating call is gated even if it bypasses the ritual — *"the orchestrator proposes, never silently restructures"* is now enforced by the gate.
- A lightweight **triage digest** (status counts, active sessions, what changed since last turn, stale streams) is injected into every board turn so the orchestrator can *notice* and propose, not just answer.

## 💬 The chat becomes a real editing surface

- **FLUX-661** — **inline per-edit diffs in the chat stream**: each edit-tool row is now an expandable diff of the file it touched (engine attaches the repo-relative path; the portal lazy-loads the hunk against the branch). Scrolling the chat becomes scrolling the ticket's edit history.
- **FLUX-674** — **paste images straight into ticket chat**: drop a screenshot into the composer and it goes to the agent inline, no detour through disk.
- **FLUX-669** — **id chips that survive the model's formatting**: backticked ids (`` `FLUX-630` ``) now render as interactive ticket chips just like bare ones, and the board prompt now requires the **full id on every mention** (no bare numbers) — fixing the case where a triage turn backticked nearly every id and only a handful chipped.

## 🗂️ Board

- **FLUX-580** — the PR deck/fold mechanic is **generalized to epics**: a subtask in the same column as its epic now **folds under the epic card** and leaves the column, unwinding on demand — exactly like PR members. A single reusable `TaskDeck` primitive now backs both the violet PR deck and the indigo epic deck, with PR membership taking precedence (no double-fold).

## 🛡️ Reliability

- **FLUX-667** — an orchestrator session that hit a **failed or parked board turn** could permanently wedge; the session no longer gets stuck and the board recovers.
