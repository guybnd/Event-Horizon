## v0.27.1

### Windows: switch to Node.js Single Executable Applications

The Windows build now uses [Node.js SEA](https://nodejs.org/api/single-executable-applications.html) instead of `@yao-pkg/pkg`. The resulting `event-horizon-win-*.exe` is a standard `node.exe` with the application bundle injected — rather than a custom runtime with a virtual filesystem. This eliminates (or significantly reduces) the `Trojan:Script/Wacatac.C!ml` false positive that Windows Defender's ML heuristic triggered against the previous build.

MCP mode is also supported in the SEA binary: `mcp-server.js` is embedded in the binary and extracted to a versioned tmpdir on first run, so agent tool connections continue to work identically.
