# Release v0.50.0

*Released 2026-06-19*

The release where **the board became a place you actually work**. v0.50.0 lands two big arcs at once: pull requests grow into a **full review-and-merge lifecycle** that lives entirely on the board, and the per-ticket **chat experience matures into a top-tier client** — live windows, a streamed activity timeline, inline diffs, and an orchestrator that dispatches work for you. Underneath, a focused **performance pass** kills the board re-render storm, and the MCP layer moves **over HTTP** so any worktree or external agent can reach the same ticket store.

## 🔀 PR review & merge, fully on the board

Two epics close here: **in-EH PR review & merge** (FLUX-555) and **PR-as-a-first-class-ticket** (FLUX-565). A pull request is now a real board citizen you can raise, review, bounce, and merge without leaving Event Horizon.

- **FLUX-569** — PR-ticket **lifecycle edges**: the final state machine for raise → review → changes-requested → merge, retiring the interim FLUX-558 surface.
- **FLUX-615 / FLUX-660** — **inline branch/PR diff review in the chat window**, now **live-updating** as commits land — review the actual diff next to the conversation instead of context-switching to GitHub.
- **FLUX-594** — PR deck polish: **inline re-review while changes-requested** and a deduped changes-requested indicator.
- **FLUX-597** — **Ready auto-PR** no longer skips when a *closed* PR already exists on the branch (the existing-PR check was state-blind).
- **FLUX-599** — post-merge **branch delete** no longer fails silently when the branch is checked out in a shared tree: it retries and surfaces the error instead of orphaning branches.
- **FLUX-585** — PR-ticket `baselineCommit` is anchored at the branch fork point, killing **phantom-revert review diffs** caused by a non-ancestor sibling baseline.

## 💬 Chat UX overhaul — the windows go top-tier

The headline UX arc (FLUX-638): turn the per-ticket chat from a functional panel into a genuinely good client.

- **FLUX-603** — **minimizable per-ticket chat popups** with thinking/response notifications, so several conversations can run at once without taking over the screen.
- **FLUX-639** — a **working strip** above the composer that consolidates every "running" indicator into one place.
- **FLUX-640** — a **streamed activity timeline** under the working strip: watch the agent's steps arrive live.
- **FLUX-641 / FLUX-653 / FLUX-665** — **inline ticket chips** in assistant markdown: launch chips, hover-title + click-to-pin action popovers (reusing the ticket action bar), and chipping of markdown-link / inline-wrapped ids — with the prompt example parameterized by project key.
- **FLUX-642** — an **empty-chat context card** showing the last agent comment / board snapshot so a fresh chat already knows where things stand.
- **FLUX-643** — **quick-reply chips** for Require Input pauses: one tap to answer a blocking question.
- **FLUX-644** — a **jump-to-bottom pill** during live turns.
- **FLUX-648 / FLUX-652** — dock tabs **tinted by ticket status**, plus card containment with bounded surfaces and a dedicated agent-status row so nothing overflows a card.
- **FLUX-622 / FLUX-623 / FLUX-635** — chat reliability: the misleading action buttons are fixed, **minimizing a composer no longer discards unsent text**, and dismissed chat cards stay dismissed across an app restart.

## 🧭 Orchestration & agent workflow

The board orchestrator stops talking and starts *acting*.

- **FLUX-606** — the orchestrator **dispatches work to tickets**: "groom FLUX-X" fires that ticket's phase-default session (fire-and-forget) in its own scope instead of doing it inline.
- **FLUX-662** — **AskUserQuestion** is supported in portal/board chat sessions: agents can ask a structured multiple-choice question and block on your answer.
- **FLUX-651** — the engine now **surfaces agents that end a working turn without a board action** (narrate-and-stop), flagging the ticket "Needs Action" so completed work never silently strands.
- **FLUX-637 / FLUX-647** — a board-chat **"Triage"** action for one-tap prioritization of the Todo column, disabled while the orchestrator turn is in flight.
- **FLUX-605** — `--permission-prompt-tool` is wired so **destructive ops route through EH approval**.

## ⚡ Board performance

A dedicated pass (FLUX-624) ends the re-render storm on a busy board.

- **FLUX-625** — the **AppContext value is memoized / split** so a single realtime tick no longer re-renders the whole tree.
- **FLUX-626** — fast-moving **SSE session state is isolated from the tasks array** (progress + activity streams no longer churn task identity).
- **FLUX-627** — `refreshWorktrees` is **decoupled from tasks identity**.
- **FLUX-628** — `buildTaskSignature`'s `JSON.stringify` is replaced with a **cheap comparison**.
- **FLUX-629** — framer-motion `layoutId` FLIP on board cards is **gated** so it doesn't fire during noisy updates.

## 🧰 MCP & engine

- **FLUX-645** — the Event Horizon **MCP server is now served over HTTP** from the engine, worktree-agnostic: any worktree session or external agent reaches the same live ticket store instead of binding to an empty per-worktree store.
- **FLUX-616** — a new **`archive_ticket` / `unarchive_ticket`** MCP tool: safe archive-by-default instead of hard delete (reversible, history preserved).
- **FLUX-600** — the installer no longer **clobbers user-customized module MCP entries** on every engine start (the serena http → stdio zig-zag is fixed).
- **FLUX-621** — the conversations bar and history **reset cleanly between sessions** instead of bleeding state across them.
