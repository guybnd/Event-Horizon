## Release v0.10.0

### Highlights

Multi-workspace support, global app settings with platform-conventional storage, and agent session safety guards.

### Features

**Multi-Workspace Switching**
- Workspace switcher dropdown in the portal header for instant project switching
- Full workspace management in Settings: add, remove, rename, browse with folder picker
- Auto-registration — opening a folder adds it to the workspace list
- Project bootstrapping — new workspaces are auto-initialized with config derived from folder name, default user from global settings, and agent skills pre-installed

**Global App Settings**
- Settings now stored at the platform-conventional location:
  - Windows: `%APPDATA%/EventHorizon/settings.json`
  - macOS: `~/Library/Application Support/EventHorizon/settings.json`
  - Linux: `~/.config/event-horizon/settings.json`
- First-boot welcome dialog with automatic migration from legacy `~/.event-horizon` path
- New "Global Settings" tab in Settings: theme, default user, preferred framework, port, card click behavior
- Global preferences persist across all workspaces

**Agent Session Guard**
- Switching workspace while agents are running shows a confirmation dialog
- Displays the number of active sessions and asks whether to stop them
- Prevents cross-project data confusion from orphaned agent processes

### Improvements

- Notifications cleared on workspace switch — no stale prompts from previous project
- Windows folder picker now uses TopMost form owner so the dialog appears in front of the browser
- `pathsEqual` handles Windows case-insensitive path comparison to prevent duplicate workspace entries
- Workspace switch fully tears down and reinitializes: caches, watchers, sync scheduler all reset cleanly

### Technical

- New modules: `engine/src/global-settings.ts`, `engine/src/bootstrap.ts`, `engine/src/routes/settings.ts`, `engine/src/routes/workspaces.ts`
- Portal: `FirstBootDialog`, `WorkspaceSwitcher`, `GlobalSection` components
- `workspace.ts` refactored to delegate persistence to `global-settings.ts`
- API endpoints: `GET/PUT /api/settings/global`, `GET /api/settings/boot-status`, `POST /api/settings/confirm-boot`, full workspace CRUD at `/api/workspaces`
