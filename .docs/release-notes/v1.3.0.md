## Release v1.3.0 — the performance overhaul, Furnace hardening, and a sturdier sync core
*Released at 2026-07-05T07:57:23.470Z*

### Highlights

- **Performance overhaul** — a full instrumentation epic (engine metrics registry, request timing, `GET /api/perf`, event-loop stall monitor, git subprocess timing, task-store rescan/watcher/SSE counters, portal-side timings + a perf debug panel) followed by the fix wave it uncovered: the `/api/tasks` list payload slimmed from ~16MB per poll with conditional GET, stale-while-revalidate on hot-poll endpoints, boot's double full-rescan deduped and watcher reloads made incremental, SSE token deltas coalesced instead of patching the store per delta, the idle board's always-on ~80fps animation pipeline stopped, and artifact iframes no longer paying Tailwind Play JIT cost when hidden.
- **Furnace hardening** — the slot gauge no longer desyncs from the physical worktree cap (igniting into guaranteed spawn failures), slot-holder undercounts and forced-refresh races under concurrent ignites fixed, the auto-takeover race that wedged the queue and leaked slots resolved, pre-spawn session failures now leave a durable trace in ticket chat, and vanished session worktrees get a clear signal. Plus: an intentional-selection contract for batch building, draft discard/edit + add/remove-ticket tools consolidated into action-based `furnace_batch`/`furnace_ticket` MCP tools, a burn-report modal, drag-and-drop queue reorder, a trigger editor, and review verdicts no longer stranding tickets when `reviewState` isn't set.
- **Sync & reliability** — conflicted flux-data merges are no longer silent-and-sticky (with auto-repair and PR-scanner stub fixes), sync/auth conflict notifications re-surface if dismissed while unresolved, `updateAgentSession` no longer races the per-ticket write lock, failed merges prompt a rebase/resolve agent session from the card, and the board no longer crashes on tickets with a missing/invalid status.
- **Operation telemetry** — slow network ops moved off the spawn/HTTP response path, an operation-telemetry layer (opId / kind / duration / outcome) over SSE, operation failures surfaced on ticket/session cards, and a dev ops console panel.
- **Guardrails & dev safety** — agent-run dev stacks can no longer hijack the live engine's ports, ticket chat can't edit files unless the ticket is In Progress, the engine lint burndown (~711 errors) landed and lint now blocks in the check gate, the packaged Windows exe 404 was fixed, and Electron windows can't restore off-screen.
- **UX & accessibility** — Esc closes windows consistently, consolidated + properly scoped focus traps, keyboard-accessible session cards, terminal improvements, a distinct-hue-per-status color scheme, and Engine Events badge/auto-scroll fixes.

See [`CHANGELOG.md`](../../CHANGELOG.md) for the curated, themed breakdown. Full ticket manifest below.
