## Release v1.5.1 — PR-card intelligence, plan-gate UX, and a worktree & packaging reliability wave
*Released at 2026-07-08T17:07:25.210Z*

### Highlights

A reliability- and polish-focused patch on top of v1.5.0, landing a wave of PR-card intelligence, plan-approval UX, and worktree/packaging hardening.

- **Smarter PR cards** — the PR deck now surfaces live GitHub CI/check status (FLUX-1315), and the board's PR reconcile runs immediately at boot instead of after a 90-second gap, so cards populate right after a restart (FLUX-1318). Nested member hover/click no longer opens duplicate tooltips (FLUX-1316), locked in by a regression test (FLUX-1322).
- **Plan-approval & gate UX** — a new "Approve & start" action approves a plan into Todo and dispatches implementation in one step (FLUX-1294); grooming-phase "changes requested" verdicts now show a board-card chip like the Ready phase (FLUX-1284); the "Start plan review" button is gated while the gate is already running (FLUX-1324); plus a verdict-UI scope decision (FLUX-1296), plan-review helper dedupe/hardening (FLUX-1306), and docs (FLUX-1295).
- **Worktree & branch reliability** — stale worktree folders that resist deletion now have their lock-holder processes reaped and orphaned dirs swept on prune (FLUX-1216, made cross-platform in FLUX-1327); a freshly-created worktree is no longer pruned before its session binds (FLUX-1305); a merged branch kept alive by the dependent-PR guard is auto-deleted once the dependency clears (FLUX-1326); and two more finish/merge stop sites disarm Temper/gate-runner first (FLUX-1304).
- **Standalone binary (SEA) packaging** — fixed a `createRequire(undefined)` crash that broke the standalone SEA binary (FLUX-1321), added a CI smoke test that boots the binary and hits `/api/health` (FLUX-1323), and bounded that smoke test's fetches with a timeout so a hung boot can't hang CI (FLUX-1325).
- **Furnace** — a same-branch-dependent follow-up plus its parent can be pulled into a standalone sequential batch that reuses the parent's branch (FLUX-1270); furnace-chat gained transcript-reset UI parity (FLUX-1221).
- **Engine correctness & skills** — `hasAppendedStatusChange`'s exact-prefix check no longer double-appends a `status_change` for stale full-history writers (FLUX-1311); the grooming skill now treats the `publish_artifact` decision as an explicit workflow step so delegated UI tickets don't skip it (FLUX-1313).
