## Release v0.8.0 — MCP Server & Multi-Agent Integration
*Released 2026-05-25*

### Highlights

This release introduces the **MCP (Model Context Protocol) server**, enabling all supported AI agents to interact with the ticket board through a standardized tool interface. Combined with schema enforcement and session reliability fixes, Event Horizon now provides a robust, validated pipeline for multi-agent ticket workflows.

---

### Features

- **MCP server for ticket operations** (FLUX-6): Full MCP tool server exposing `get_ticket`, `list_tickets`, `create_ticket`, `update_ticket`, `change_status`, `add_comment`, `log_progress`, `finish_ticket`, `create_subtask`, and `get_board_config`. Agents no longer need to edit `.flux/*.md` files directly.
- **Notification panel with health checks** (FLUX-302): Real-time notification system with update indicator, health check integration on session completion, and open-url action support.
- **Ticket schema enforcement** (FLUX-289): Engine validates ticket structure on read and write — malformed frontmatter is rejected with actionable errors instead of silently corrupting state.
- **Subtask creation endpoint** (FLUX-278): `POST /api/tasks/:parentId/subtasks` atomically creates a child ticket and links it to the parent's `subtasks` array.
- **Auto-create ticket files from inline subtasks** (FLUX-277): Engine detects inline subtask objects and materializes them as proper ticket files automatically.
- **First-run experience improvements** (FLUX-267): Fixed default config generation and added user name prompt during initial setup.

### Bug Fixes

- **Agent session persistence** (FLUX-274): Agents now correctly persist ticket updates after session completion — previously, updates made during a session could be lost.
- **Agent session history drops** (FLUX-306): Fixed sessions being silently dropped from ticket history due to prompt misconfiguration.
- **Legacy status_change normalization** (FLUX-287): Engine auto-normalizes `oldStatus/newStatus` shapes to canonical `from/to` format on read.
- **Portal static serving on Node 26** (FLUX-293): Fixed ESM `require()` fallback that broke dev-mode static file serving.
- **Windows console popups** (FLUX-279): Hidden `cmd.exe` console windows that flashed when spawning git/PowerShell child processes.

### Documentation & Skills

- **Orchestrator skill examples** (FLUX-288): Pinned correct vs. incorrect history-entry shapes with explicit WRONG/RIGHT annotations to prevent agent mistakes.

### Multi-Agent Integration

- Verified MCP server compatibility with **Claude** (FLUX-294), **Gemini** (FLUX-295), and **Copilot CLI** (FLUX-296). All three agent frameworks can now create, update, and close tickets through the standardized MCP interface.

---

### Upgrade Notes

- The MCP server runs on the engine process — no additional setup required beyond the standard `npm run dev`.
- Agents configured with the Event Horizon MCP will automatically use tools instead of direct file edits.
- Legacy `oldStatus/newStatus` history entries are auto-migrated on read; no manual action needed.
