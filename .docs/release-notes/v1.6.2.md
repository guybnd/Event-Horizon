## Release v1.6.2 — a buttery-smooth chat dock, plus board, desktop, and session polish
*Released at 2026-07-14T05:14:36.312Z*

### Highlights

A performance-and-polish patch. The headline is the chat dock: under real fleet load (10+ concurrent agents) opening, minimizing, and streaming into chat windows now stays smooth instead of freezing-then-snapping.

- **Snappier chat dock** — the open/minimize spring no longer competes with the window's own mount/unmount work: heavy content is deferred past the open animation, minimize gets a cheap exit, and the window root is compositor-promoted (FLUX-1418). SSE-driven re-renders are buffered while the spring is in flight and flushed on settle, so live agent output can't hitch the animation (FLUX-1420). Opening a scratch chat is near-instant again — the blocking `git fetch` and a rename round-trip were removed from the mint hot path (FLUX-1417). And the board's ambient glow/pulse effects were converted from paint-invalidating `box-shadow` loops to compositor-only opacity, cutting per-frame raster cost (FLUX-1419).
- **Board & prompt UX** — a subtask with a live agent session now stays expanded under its epic instead of hiding behind a collapsed peek (FLUX-1422); the Fast-path button holds its busy state continuously through the post-launch refresh, so a click never looks like a no-op (FLUX-1423); inline prompt questions no longer overflow the chat window (FLUX-1413); the "recommended" option is highlighted in multi-select prompts (FLUX-1409); and parked sessions no longer masquerade as "agent running" on cards (FLUX-1414).
- **Desktop & packaging** — the Electron desktop build's skill install no longer always fails (skill-source path resolution fix, FLUX-1416), and the packaged app now tells "gh not found" apart from "gh not authenticated" instead of misreporting one as the other (FLUX-1402).
- **Session & worktree reliability** — a stuck `running` session can no longer be double-dispatched by a POST retry (FLUX-1392), and worktree reclamation deduplicates its board notifications and fixes a misleading clean-remove-failure message (FLUX-1411).
- **Cost & MCP efficiency** — MCP tool descriptions were trimmed to ≤50 words each, and tool-schema weight is now measured in the context-budget metrics so schema bloat is visible (FLUX-1386, FLUX-1376).
