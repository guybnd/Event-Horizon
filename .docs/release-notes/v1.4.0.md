## Release v1.4.0 — the persona overhaul: server-composed prompts, new specialist agents, and structured handoffs
*Released at 2026-07-06T01:13:25.203Z*

### Highlights

- **Persona system overhaul** — persona prompts are now composed server-side from a shared phase contract plus a per-persona lens instead of hand-maintained parallel prompt files (FLUX-1170), backed by a new dedicated review-phase skill module (FLUX-1171) and a dedup pass hoisting shared invariants into the orchestrator module (FLUX-1172). Long-standing prompt drift — commit-before-Ready, the sole-reviewer `reviewState` contract, and the diff base — was fixed at the source (FLUX-1169).
- **New specialist personas** — the roster grew four specialists: the **Furnace Operator ("Smelter")**, a furnace-owning chat persona that curates, ignites, and shepherds batches (FLUX-1175); a **DRY / Reuse & Simplicity reviewer** (FLUX-1173); the **Regrounder**, which executes the FLUX-1048 reground as a delegable step (FLUX-1174); and the **Epic Decomposer**, which splits L/XL tickets into Furnace-sized subtasks (FLUX-1176). The roster was also consolidated: Coordinator merged into Supervisor (FLUX-1177), and the finalize personas folded into a single **Shipper** (Ticket Curator absorbed, Committer + PR Merger merged — FLUX-1178), with a retired-id alias fix so old ids can't shadow same-named custom personas (FLUX-1198).
- **Structured agent workflow** — ticket completion is now machine-readable: a structured Ready/finish handoff payload carrying changed files, validation results, decisions, and residual risk (FLUX-1147); **acceptance criteria & Definition of Done** as first-class ticket fields (FLUX-1148); an append-only, agent-consumable release-notes `INDEX.md` so agents can cheaply answer "was this already done?" (FLUX-1151); and a dynamic board-health triage prompt feeding `propose_board_rebase` (FLUX-966).
- **Perf follow-through** — the buttery-smooth portal epic closed (FLUX-1135): heavy clicks profiled on a prod build and the top offender fixed (FLUX-1141), the FurnaceDrawer memoized so polls no longer re-render every batch and row (FLUX-1196), event-loop-stall warnings confirmed gone after the yield fix (FLUX-1190), the board tee gap + per-turn git subprocess cost on resume addressed (FLUX-1182), and a regression test locking in the `.eh-idle` ambient-animation pause (FLUX-1191).
- **Furnace & engine hardening** — the Ready-transition guard no longer misattributes main-checkout dirty state to a ticket's isolated worktree (FLUX-1121); concurrent forced `refreshWorktreePool()` calls can't clobber each other's in-flight reference (FLUX-1192); a cache-invalidation race in `isRegisteredWorktree` narrowed (FLUX-1195); the trigger editor no longer arms triggers on parked/done batches that can never fire (FLUX-1181); and `excludeSerenaOverrideFromGitStatus` is guarded against concurrent writes to `info/exclude` (FLUX-1162).
- **Agent boundaries** — the "ticket chat can't edit files unless In Progress" gate, previously Claude-only, now covers Copilot and Gemini ticket chat too (FLUX-1123), with wiring-level regression tests (FLUX-1193).
- **Epics closed** — the MCP token-footprint & agent-context optimization epic (FLUX-477) and the Restore Stability epic (FLUX-996) both reached Done, closing out their umbrella tracking.
- **Tests & polish** — the `useFocusTrap` Escape gap for non-modal traps fixed (FLUX-1118) plus ESC typing-guard polish with test coverage (FLUX-1180), CardCommentBadge gating tests (FLUX-1165), the task-store-watcher incremental-burst test deflaked (FLUX-1194), and FurnaceReportModal cleanup (FLUX-1159).

See [`CHANGELOG.md`](../../CHANGELOG.md) for the curated, themed breakdown. Full ticket manifest below.
