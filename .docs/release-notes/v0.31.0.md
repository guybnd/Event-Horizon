# Release v0.31.0

*Released 2026-06-18*

The release where **the board started talking back**. v0.31.0 introduces **EH-as-client** — every ticket becomes its own live, resumable Claude chat, tied together by an always-on board orchestrator — and matures the **PR-as-ticket** model into a full pull-request lifecycle on the board. Underneath, a deep **board-performance** pass makes a busy board (and dragging on it) smooth again, alongside a long tail of worktree, PR, and engine reliability fixes.

## 💬 EH-as-client — tickets are live chats

The headline direction: stop treating a ticket as a static card and start treating it as a **conversation you can pick up at any time**.

- **FLUX-602** — the spike that proves it: a live, **resumable** agent session bound to one ticket, with the full `event-horizon` MCP toolset. A finished turn parks the session (resumable) instead of ending it, so the next message continues the *same* conversation with full memory — markdown for the ticket, JSONL for the raw transcript.
- **FLUX-604** — an always-on **board orchestrator**: a board-level chat (not tied to any ticket) that can reason across the whole board and **dispatch** work — "groom FLUX-X" starts that ticket's session in its own scope rather than doing it inline.
- **FLUX-607** — a proper **chat dock**: a Windows-taskbar-style strip pinned bottom, one tab per live/recent chat, at-a-glance state (working / needs-input / finished), with floating, movable chat windows.
- **FLUX-610** — a **phase-aware action bar** in the chat: engine actions (status moves, finish) run as free, instant REST calls, distinct from deliberate, tokenized agent dispatch.
- **FLUX-620** — an inline **ticket context header** in each chat window (title, status, branch/worktree) so a chat always knows what it's about.

## ⚡ Board performance

A busy board with active agent sessions had gotten sluggish — and dragging a ticket could crawl to ~1fps. This release rebuilds the hot paths:

- The board derivation chain (filter → sort → per-column bucketing) is **memoized** and bucketed once into a map, instead of re-running on every render and every realtime tick.
- Live chat surfaces moved **off polling onto the existing SSE stream** (parked chats now poll zero).
- **Dragging is fixed at the root:** cards no longer re-render while you drag — a thin sortable wrapper carries the motion while the heavy card body is frozen ("cardboard cutout"), and the board stops mutating mid-drag. The drag-start hang and the 1fps are gone.

## 🔀 PR-as-ticket — the full lifecycle on the board

Pull requests are now first-class board citizens, from raise to merge to retry.

- **FLUX-566 / FLUX-567 / FLUX-568** — PR-as-ticket **P1–P3**: the PR entity with `gh`-driven sync, the board **deck** (members fold into their PR card, fold/unwind), and the PR surface + actions.
- **FLUX-556 / FLUX-557 / FLUX-558** — engine PR state + raise/merge endpoints, post-merge worktree cleanup + master sync, and the Open-PRs swimlane / glow / Raise-PR action.
- **FLUX-559** — review flow: send-for-review and bounce-back-to-In-Progress.
- **FLUX-561 / FLUX-563** — hardened PR endpoints (error codes, merge-loop atomicity) and a commit-before-Ready guard reconciled with the skills.
- **FLUX-578 / FLUX-587 / FLUX-588 / FLUX-593 / FLUX-598** — create-PR-if-missing before merge, merged-PR state no longer shows OPEN in Done, reopening a merged-PR ticket sticks, retry-a-merged-PR spawns a linked ticket, and the reconciler stops mangling PR tickets.

## 📋 Uncommitted-changes panel

See and commit your working tree without leaving the board.

- **FLUX-535** — a green/yellow/red **stoplight** of uncommitted file count on the board.
- **FLUX-544 / FLUX-554** — a dropdown panel with **per-file diffs** and **commit-selected-files**.
- **FLUX-564** — per-section select-all, collapsible sections, and a cross-worktree count.

## 🛡️ Worktree & reliability

- **FLUX-571** — worktree sessions not spawned by EH can now reach the ticket store (MCP no longer binds to the empty worktree store).
- **FLUX-574 / FLUX-577** — `finish_ticket` on a worktree ticket no longer errors on branch-delete; deleting a ticket tears down its worktree/branch/sessions instead of orphaning them.
- **FLUX-570** — blocking finish/finalize decisions can no longer die silently in the session log — they surface as a notification.

## 🔒 Permissions & settings

- **FLUX-523** — "Project Folder" consolidated into Configured Workspaces in Settings.
