## Release v1.6.4 — multi-engine sync rebuilt on push-as-CAS
*Released at 2026-07-15T06:12:47.186Z*

### Highlights

A sync-reliability release: the flux-data store no longer 3-way-merges ticket files between engines, plus fixes for shared batch branches and unattended agent sessions.

- **Push-as-CAS with idempotent command replay** — syncing the flux-data store between two engines no longer relies on textual git merges of ticket files (the source of the recent conflict-marker corruptions). Every human/agent mutation is journaled durably as an intent before it's applied; the engine then commits and pushes, treating the remote as an atomic compare-and-swap. If the push loses a race, the engine resets to the remote head and replays its un-flushed intents through the normal handlers — so a duplicate "advance to Done" becomes a clean no-op instead of a conflict marker or a second Done event (FLUX-1428).
- **Sync protocol gate** — the store now carries a `sync-protocol` version marker. An engine that's too old for the store's protocol goes read-only on sync (the local board keeps working) and shows a distinct "this store requires a newer engine" status, so a stale engine physically cannot write corruption into a shared store (FLUX-1426).
- **PR-mirror cards resolve conflicts per field** — when both engines rewrite the same PR card, GitHub-owned scalars (`prState`, `ciStatus`, `reviewDecision`, …) are simply re-polled from GitHub, while EH-owned state — review comments, a send-for-review status, a merge-conflict swimlane — is unioned and preserved. PR cards never reach manual conflict resolution anymore, and never silently drop your review-send or comments (FLUX-1427).
- **Shared batch branches no longer torn down under live siblings** — when one ticket's PR on a shared (sequential-batch) branch merged, post-merge cleanup used to stop every sibling's session and rip out the worktree even if a sibling was mid-work. Cleanup now defers until no ticket on the branch has a live session (FLUX-1433).
- **No more empty "I'll wait for X" promises** — an unattended chat-phase session that ends its turn promising to wait for something, with no real wakeup armed, is now surfaced as needing action instead of silently sitting idle forever (FLUX-1432).
- **Leaner worker sessions** — worker delegate sessions (reviewers, scouts, interrogators) now get a minimal role-scoped Event Horizon toolset instead of all ~27 board tool schemas, cutting per-session token overhead; lead and sole-reviewer sessions keep the full write set so the end-of-turn action contract is never broken (FLUX-1385).
