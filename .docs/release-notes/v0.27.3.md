## v0.27.3

Fixes the version string reported by the SEA (Windows) binary.

### Fix

- **Binary reported its version as `0.0.0`** and therefore always showed an "update available" prompt. `getLocalVersion()` read `package.json` from disk relative to the executable, but no `package.json` sits next to a SEA binary, so the read failed and fell back to `0.0.0`. The version is now inlined into the bundle at build time via esbuild `define` (`__EH_VERSION__`), so it is correct regardless of packaging mode; dev (`tsx`) still falls back to reading `package.json`. The SEA manifest reuses the same value so the two can't drift.
