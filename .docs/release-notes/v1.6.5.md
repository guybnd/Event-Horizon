## Release v1.6.5 — desktop dialog sweep + unattended-session reliability
*Released at 2026-07-16T06:00:48.212Z*

### Highlights

A patch release in two threads: the portal no longer uses native browser dialogs anywhere (they throw or silently no-op in the Electron desktop app), and a batch of fixes for unattended agent sessions that could stall, park, or swallow input without telling anyone.

**Desktop app / portal UX**

- **Native dialogs are gone portal-wide** — every `alert()`/`confirm()`/`prompt()` call is replaced with styled in-app surfaces: a shared `useConfirm`/`useNotify` pair for confirmations and notices (FLUX-1459), a `PromptModal` for link insertion in the docs and description editors (FLUX-1457), and ticket actions decoupled from `window.prompt`/`alert` entirely (FLUX-1359). In the Electron desktop app the native versions throw or vanish; now every dialog renders consistently everywhere.
- **Closing the desktop app with unsaved doc edits works again** — a dirty doc's `beforeunload` handler silently blocked window close in Electron with no dialog ever shown; the app now surfaces a proper unsaved-changes prompt instead of appearing frozen (FLUX-1458).
- **Chat text streams smoothly** — streaming assistant text renders with polished animation instead of chunky repaints (FLUX-1439).
- **Guided annotation on grooming artifacts** — you can annotate a plan-time mockup directly and the annotations feed back into grooming as structured guidance (FLUX-1440).

**Agent-session reliability**

- **Finish is never silently swallowed** — pressing Finish while a stale parked session was still registered routed the command into a session that would never act on it; Finish now routes past stale waiting-input sessions (FLUX-1456).
- **Stalled review sessions self-heal** — a session that ended its turn waiting on background work without a verdict is caught and resumed, and Temper auto-retries instead of leaving the ticket in limbo (FLUX-1437).
- **Chat input reaches the right session** — typing into a ticket chat with no explicit target resumes the supervisor lead (lead > solo > delegate), not whichever delegate happened to spawn last (FLUX-1435). The parked-ticket backstop also no longer exempts supervisor leads, so a parked orchestrator gets flagged like any other session (FLUX-1436), and the needs-action flag is raised again on the wake-resume crash path (FLUX-1393).
- **Worker toolset scoping rebuilt as a deny-list** — the v2 fix for the per-role toolset regressions: workers get their minimal toolset without losing tools they legitimately need (FLUX-1434).
- **Furnace resilience** — recreating a worktree over a locked empty husk retries instead of hard-parking the ticket on EBUSY (FLUX-1442), scratch tickets must be promoted to a real groomed Todo before implementation starts (FLUX-1443), and very long prompts no longer kill session spawn with ENAMETOOLONG — the prompt is delivered over stdin (FLUX-1444).

**Process**

- **Ceremony diet** — mandated comment/summary ceremony now scales with ticket effort, so an XS fix doesn't demand an XL write-up (FLUX-1382).
