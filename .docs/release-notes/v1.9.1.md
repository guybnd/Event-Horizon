## Release v1.9.1
*Released at 2026-07-23*

> A small reliability patch: closes a silent MCP-permission wedge and an immediate-feedback gap on the board-rebase propose path.

### Engine & agent hygiene

**No more silently denied ticket tools.** A restrictive local Claude Code permission default (e.g. `dontAsk` mode) could silently deny every event-horizon MCP tool call from an unattended/orchestrator session — no prompt (there's no user to ask), no error anywhere but the denied session's own transcript. The workspace installer now bakes an explicit `mcp__event-horizon` allow rule into the project's committed `.claude/settings.json` at install time, gated by a new per-CLI capability flag rather than a framework literal. Gemini CLI gets the equivalent fix (`trust: true` on its MCP server entry); Copilot CLI has no project-committable equivalent, which is now documented rather than silently gapped.

**Board-rebase validation moved earlier.** Promoting a scratch ticket into Todo could fail with "fromSeq and toSeq are required" only after a human clicked "Apply approved" in the portal — `propose_board_rebase` happily parked an incomplete `promote` item with no feedback to the proposing agent. Per-kind required-field checks now run inside `proposeBoardRebase()` itself, so a bad item is rejected immediately, in the same turn, with an actionable message (FLUX-1607).
