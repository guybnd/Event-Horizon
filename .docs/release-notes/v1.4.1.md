## Release v1.4.1 — Smelter follow-ups, HITL routing fix, and Furnace patch wave
*Released at 2026-07-06T05:56:42.731Z*

### Highlights

A small patch on the heels of v1.4.0, landing the follow-up wave from the Smelter persona's first day in the field plus a high-priority HITL routing fix.

- **Smelter follow-ups** — the Furnace Operator persona shipped in v1.4.0 and immediately earned a fix wave: its chat no longer overrides the orchestrator chat and instead lives on its own dedicated surface (FLUX-1209, with the placement decision tracked in FLUX-1212), opening its chat no longer auto-sends a pre-recorded message instead of a blank composer (FLUX-1211), and it now recognizes when its own furnace tickets are already merged (FLUX-1210).
- **HITL prompt routing** — `ask_user_question` / `permission_prompt` raised from dedicated ticket sessions over shared HTTP MCP no longer misroute to the board orchestrator; prompts land on the ticket that asked (FLUX-1213).
- **Worktree hygiene** — grooming-phase sessions no longer create a branch+worktree they never use (and orphaned ones are reclaimable), stopping a slow slot-pool leak (FLUX-1214); a health signal now surfaces when Furnace worktree slots are exhausted with no batch burning (FLUX-1217).
- **Sequential Furnace batches** — the shared PR's title/body and `batch.prs` now reflect every ticket in the batch, not just the first (FLUX-1223).
- **Perf follow-ups** — ChatDock wide-context, TaskModalPopupView skeleton gap, and Board filter deferral fixes from the FLUX-1141 profiling work (FLUX-1200).

See [`CHANGELOG.md`](../../CHANGELOG.md) for the curated breakdown. Full ticket manifest below.
