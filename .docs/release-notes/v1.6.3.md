## Release v1.6.3 — the Windows desktop terminal works again
*Released at 2026-07-14T08:55:26.508Z*

### Highlights

A focused fix for the desktop app's integrated terminal on Windows.

- **Windows desktop terminal fixed** — opening a terminal in the packaged desktop app on Windows failed outright: the engine bundle shipped no `node-pty` native binary, so the terminal threw `Failed to load native module: conpty.node`. Staging the binary then surfaced a second, worse failure — because `node-pty` had been inlined into the engine bundle, its ConPTY helper (a Worker script plus a forked agent) wasn't present on disk, so the first terminal open crashed the whole engine and took the board down with it. `node-pty` now ships as a proper external package next to the engine, with its native binaries and helper scripts intact, so terminals open, stream output, and close cleanly (FLUX-1425).
- **Release tooling** — `flux:release` now also bumps `electron/package.json` and the root `package-lock.json`, dropping a manual step from every release cut (FLUX-1424).
